================================================================================
PROCEDIMIENTOS ALMACENADOS, USUARIOS MySQL Y PRIVILEGIOS 
================================================================================

La conexión PDO usa el usuario MySQL asociado a $_SESSION['tipo_usuario'] en
public/src/data/conexion.php. Si no hay sesión válida, el rol es 'invitado'.

================================================================================
0. PERMISOS PHP (seguridad.php) POR ROL — acciones verificarPermisos()
================================================================================
  administrador → ver_reportes, crear_lugar, ver_lugar, editar_lugar, eliminar_lugar
  organizadora  → crear_evento, editar_evento, ver_lugar, eliminar_evento, ver_eventos
  agencia       → crear_paquete, ver_lugar, ver_paquete, editar_paquete, eliminar_paquete
  usuario       → ver_eventos, ver_lugar, registrarse_evento, registrarse_paquete

Nota: muchos Logic NO llaman a verificarPermisos() (lecturas públicas). Eso
implica que invitado u otro rol pueden disparar esos SP si conocen la URL; el
usuario MySQL usado sigue siendo el de la sesión actual (o invitado).

2. PROCEDIMIENTOS EN DAO SIN USAR (Deben ser 7)
================================================================================
  sp_alta_agencia, sp_actualizar_agencia, sp_baja_agencia
    → métodos altaAgencia / actualizarAgencia / eliminarAgencia en agenciaDAO;
      ningún archivo en Logic/ las invoca (solo validarAgencia y getAgenciaPorID).

  sp_filtrar_eventos_por_lugar
    → eventosDAO::filtrarEventosPorLugar; solo referencia en organizerController
      dentro de un bloque /* comentado */.

  sp_get_all_viajes, sp_get_viaje_por_id, sp_crear_viaje, sp_actualizar_viaje,
  sp_eliminar_viaje
    → métodos en viajesDAO; Logic solo usa getViajesPorCliente y cancelarViaje
      (este último usa sp_actualizar_estado_viaje). CrearViajeLogic usa
      CrearViajeDAO → sp_crear_viaje (mismo SP, otra clase).

4.2 arc_admin (administrador)
--------------------------------------------------------------------------------
Incluye todo lo que el invitado puede ejecutar por URLs sin protección, MÁS el
CRUD de lugares vía lugarLogic (verificarPermisos ver_lugar / crear_lugar /
editar_lugar / eliminar_lugar):

  sp_crear_lugar
  sp_actualizar_imagen_lugar
  sp_actualizar_lugar
  sp_eliminar_lugar

y las lecturas de lugar ya listadas (sp_get_lugares, sp_get_lugar_por_id).

NO puede ejecutar vía PHP (verificarPermisos lo impide): POST de eventController
(crear/editar/eliminar evento), POST de packageAgency (paquetes), crear lugar
como organizadora no aplica. tampoco tiene permisos de paquete en seguridad.php.

4.5 arc_cliente (usuario / cliente)
--------------------------------------------------------------------------------
Incluye las mismas lecturas públicas que en 4.1 si el cliente navega esas URLs
(con sesión usuario la conexión es arc_cliente).

Flujos propios de cliente (reservas y viajes):

  sp_cancelar_reservacion

  sp_crear_viaje
